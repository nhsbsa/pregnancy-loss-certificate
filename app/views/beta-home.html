Second parent IDV

Miro board: Second parent identity verification options Miro .

Scenario 1

Main applicant does their own and second parents ID Verification, where a OTP is sent to second parent to verify their details, implied consent given. (Both parents need to be together or on the phone (or other) to complete the single synchronous journey.)

Use date 27/3/2007 for date of birth, any other dates will guide you to the kickout screen

Scenario 1 start

Scenario 2

Main applicant does their own ID verification and NOT do second parents ID Verification. Second parent completes a prompted journey where they verify who they are, ultimately giving consent. This is an asynchronous journey! (2nd parent gets a notification via email or SMS that contains a link to start the journey.)

Use date 27/3/2007 for date of birth, any other dates will guide you to the kickout screen

Scenario 3

Main applicant provides second parent with a reference code to enter into second parent start page. The second parent completes the journey, proving their identity and gives explicit consent. (The first parent receives a journey/reference code/identifier. They give that to the second parent to complete the consent journey via a new start page. This journey will validate the user’s name.) This is an asynchronous journey!

Use date 27/3/2007 for date of birth, any other dates will guide you to the kickout screen

Scenario 2 & 3 start
